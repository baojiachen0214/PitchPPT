## V4: 平均配额算法

### 核心概念

```
A = 基准体积（删除图片后的PPT大小）
C = 目标大小限制
N = 幻灯片页数
每页配额 = (C - A) / N
```

### 处理流程

1. **计算基准体积 (A)**
   - 从PPT中移除所有图片内容
   - 保存并测量文件大小
   - 这代表非图片元素的体积

2. **检查边界条件**
   - 如果H=480px仍超过目标 → 所有页面使用最低配置
   - 如果H=4000px仍未达到目标 → 所有页面使用最高配置

3. **逐页优化**
   - 从默认高度开始二分搜索
   - 使用"正负突变"作为停止条件
   - 每页独立优化

4. **精细调整**
   - 多尺度调整：50px → 20px → 10px → 5px → 1px
   - 确保最终误差 < 2%

### 优势
- 简单稳定
- 结果可预测
- 处理速度快

### 局限性
- 不考虑内容复杂度差异
- 可能在简单页面上浪费配额
- 复杂页面可能配额不足

## V5: 双轮优化算法

### 核心概念

第一轮测试实际压缩率，第二轮根据结果调整配额。

### 处理流程

1. **第一轮：测试**
   - 在参考高度（如1080px）导出所有页面
   - 记录每页的实际文件大小
   - 计算压缩率

2. **配额重新计算**
   ```
   对于每页：
   调整后配额 = 基础配额 × (页面压缩率 / 平均压缩率)
   ```

3. **第二轮：优化**
   - 使用调整后的配额优化每页
   - 应用与V4相同的优化逻辑

4. **最终微调**
   - 验证实际文件大小
   - 如需要则进行多尺度调整
   - 如超过限制则回退到最后有效配置

### 优势
- 基于实际结果更好地分配配额
- 比V4精度更高
- 处理变化的内容复杂度

### 局限性
- 需要两次遍历（较慢）
- 第一轮可能耗时较长

---

## V6: 迭代优化算法

### 核心概念

根据在参考高度测量的内容复杂度动态分配配额。

### 处理流程

1. **复杂度评估**
   - 在参考高度（1080px）导出每页
   - 文件大小直接反映内容复杂度
   - 文件越大 = 内容越复杂

2. **比例分配**
   ```
   总可用配额 = C - A
   对于每页：
   页面配额 = 总可用配额 × (页面大小 / 所有页面大小之和)
   ```

3. **迭代优化**
   - 使用计算配额进行初始优化
   - 测量实际结果
   - 根据误差调整配额
   - 重复直到收敛

4. **最终组装**
   - 组合所有优化后的页面
   - 验证总大小
   - 如需要则进行最终微调

### 优势
- 最准确的配额分配
- 适应内容复杂度
- 复杂页面获得最佳图像质量

### 局限性
- 处理时间最慢
- 计算量最大

## 共同特性

### 多尺度微调

三种算法使用相同的微调策略：

```python
scales = [
    (50, 1),   # 50px调整，1次迭代
    (20, 1),   # 20px调整，1次迭代
    (10, 1),   # 10px调整，1次迭代
    (5, 1),    # 5px调整，1次迭代
    (1, 3)     # 1px调整，3次迭代
]
```

### "正负突变"停止条件

二分搜索期间：
- 跟踪当前大小是高于还是低于目标
- 当方向改变时停止（低于→高于 或 高于→低于）
- 使用最后一个"低于目标"的配置

### 回退机制

如果最终大小超过用户限制：
1. 跟踪所有有效配置（大小 ≤ 目标）
2. 如果最终超过限制，回退到最后有效配置
3. 确保最终结果始终满足用户要求

## 性能特征

### 处理时间（50页PPT）

| 算法 | 平均时间 | 精度 |
|------|----------|------|
| V4 | 2-3分钟 | ±3-5% |
| V5 | 4-6分钟 | ±2-3% |
| V6 | 6-10分钟 | ±1-2% |

### 内存使用

- 峰值内存：~500MB（处理大PPT时）
- 临时文件：~2倍目标大小
- 清理：完成/出错时自动清理

## 使用建议

### 何时使用每种算法

**使用V4当：**
- 页面内容复杂度相似
- 速度是优先考虑的
- 需要简单、可预测的结果

**使用V5当：**
- 页面间内容差异显著
- 需要速度和精度的平衡
- 首次使用用户

**使用V6当：**
- 需要最大精度
- 页面复杂度差异很大
- 时间不是限制因素
- 最终比赛提交
