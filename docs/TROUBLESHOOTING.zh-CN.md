## 安装问题

### 问题："ModuleNotFoundError: No module named 'src'"

**原因：** 从错误目录运行文件。

**解决方案：**
```bash
# 正确方式
cd d:\Pythoncode\PitchPPT
python src\main.py

# 错误方式（不要这样做）
cd d:\Pythoncode\PitchPPT\src
python main.py
```

### 问题："ImportError: No module named 'PyQt5'"

**原因：** 未安装依赖。

**解决方案：**
```bash
pip install -r requirements.txt
```

### 问题："ImportError: No module named 'win32com'"

**原因：** 未安装pywin32。

**解决方案：**
```bash
pip install pywin32
```

## PowerPoint问题

### 问题："PowerPoint初始化失败"

**症状：**
- 关于PowerPoint COM接口的错误信息
- 程序启动时崩溃

**解决方案：**

1. **验证PowerPoint安装**
   ```
   - 手动打开PowerPoint
   - 检查是否能无错误打开
   - 尝试创建新演示文稿
   ```

2. **修复Office安装**
   ```
   Windows设置 → 应用 → Microsoft Office → 修改 → 修复
   ```

3. **以管理员身份运行**
   ```
   右键PitchPPT.exe → 以管理员身份运行
   ```

4. **检查PowerPoint版本**
   - 最低要求：PowerPoint 2016
   - 推荐：PowerPoint 2019/365

### 问题："PowerPoint正忙"

**原因：** 另一个PowerPoint实例正在运行。

**解决方案：**
1. 关闭所有PowerPoint窗口
2. 检查任务管理器中的POWERPNT.EXE
3. 结束任何残留的PowerPoint进程
4. 重启PitchPPT

## 转换问题

### 问题："导出失败"

**常见原因和解决方案：**

1. **源文件损坏**
   ```
   - 在PowerPoint中打开PPT
   - 另存为新文件
   - 尝试转换新文件
   ```

2. **磁盘空间不足**
   ```
   - 检查可用空间（需要2倍目标大小）
   - 清理临时文件
   - 更改输出目录
   ```

3. **权限问题**
   ```
   - 以管理员身份运行
   - 更改输出到用户目录（桌面、文档）
   - 检查文件夹权限
   ```

4. **文件名中的特殊字符**
   ```
   - 移除特殊字符：/ \ : * ? " < > |
   - 使用简单的ASCII文件名
   ```

### 问题："处理卡在X%"

**诊断：**
1. 检查任务管理器中PowerPoint是否仍在活动
2. 查看日志文件了解详情
3. 检查文件是否异常大

**解决方案：**

1. **等待更长时间**
   - 大文件（100+页）可能需要10分钟以上
   - 复杂动画会减慢处理速度

2. **降低复杂度**
   - 移除不必要的动画
   - 压缩嵌入的视频
   - 简化复杂图形

3. **重启并重试**
   ```
   - 停止进程
   - 重启PitchPPT
   - 尝试不同设置
   ```

### 问题："输出文件大小超过目标"

**原因：**
1. 算法无法充分压缩
2. 目标大小太接近限制
3. 文件包含不可压缩元素

**解决方案：**

1. **使用V6算法**
   - 最精确的大小控制
   - 更好的配额分配

2. **降低目标大小**
   ```
   如果限制是50MB，将目标设为45-47MB
   ```

3. **检查大元素**
   ```
   - 嵌入的视频（转换为链接）
   - 高分辨率图像（单独压缩）
   - 嵌入字体（子集化字体）
   ```

4. **降低质量设置**
   ```
   - 降低DPI（尝试96-150）
   - 降低图像质量滑块
   ```

## 质量问题

### 问题："图像看起来模糊"

**解决方案：**

1. **增加DPI**
   ```
   设置 → 图像质量 → DPI: 200-300
   ```

2. **使用V6算法**
   - 更好地分配复杂页面
   - 在需要的地方保留细节

3. **增加目标大小**
   ```
   如果竞赛允许，使用更高的限制
   ```

4. **检查原始质量**
   ```
   - 原始图像可能质量低
   - 替换为更高分辨率版本
   ```

### 问题："文字看起来像素化"

**原因：** 文字在低分辨率下渲染为图像。

**解决方案：**

1. **保持文字为文字**
   ```
   - 不要将文字转换为图像
   - 使用PowerPoint的原生文本框
   ```

2. **增加导出分辨率**
   ```
   更高的DPI保持文字清晰度
   ```

3. **使用矢量图形**
   ```
   - 使用SVG标志代替PNG
   - 保持图表为可编辑对象
   ```

## UI问题

### 问题："窗口显示不正确"

**解决方案：**

1. **检查显示缩放**
   ```
   Windows设置 → 显示 → 缩放与布局
   - 尝试100%或125%
   ```

2. **更新显卡驱动**
   ```
   设备管理器 → 显示适配器 → 更新驱动
   ```

3. **以兼容模式运行**
   ```
   右键exe → 属性 → 兼容性
   ```

### 问题："按钮无响应"

**原因：**
- 后台进程阻塞UI
- 线程死锁

**解决方案：**

1. **等待当前操作**
   - 某些操作会暂时阻塞UI
   - 检查进度条

2. **重启应用**
   ```
   关闭并重新打开PitchPPT
   ```

3. **检查日志**
   ```
   logs/pitchppt_*.log 查看错误
   ```

## 性能问题

### 问题："处理非常慢"

**优化提示：**

1. **关闭其他应用**
   - 释放RAM和CPU
   - 特别是其他Office应用

2. **使用SSD作为临时文件**
   ```
   将临时目录设置为SSD驱动器
   ```

3. **降低PPT复杂度**
   ```
   - 移除未使用的母版幻灯片
   - 处理前压缩图像
   - 如果不需要删除隐藏幻灯片
   ```

4. **选择更快的算法**
   ```
   V4最快，V6最慢
   ```

### 问题："内存使用高"

**正常行为：**
- 峰值使用：大文件500MB-1GB
- 处理期间临时使用

**如果过高：**

1. **分批处理**
   ```
   不要一次处理100+文件
   ```

2. **会话间重启**
   ```
   关闭并重新打开以释放内存
   ```

3. **检查内存泄漏**
   ```
   在任务管理器中监控内存
   如果持续增加请报告
   ```

## 错误代码

### 错误："优化失败：基准体积A超过目标"

**含义：** 您的PPT不含图片时已大于目标。

**解决方案：**
1. 增加目标大小
2. 移除非图片内容
3. 压缩嵌入对象

### 错误："导出失败：无效文件格式"

**含义：** 源文件损坏或不是有效的PPT。

**解决方案：**
1. 在PowerPoint中打开并重新保存
2. 导出到新PPTX文件
3. 检查文件扩展名

### 错误："权限被拒绝"

**含义：** 无法读取源文件或写入目标。

**解决方案：**
1. 以管理员身份运行
2. 更改输出目录
3. 检查文件是否在其他地方打开
